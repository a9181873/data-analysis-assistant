## 產品亮點與優勢

### 地端 + 雲端雙模式
預設 100% 本機離線執行（Ollama），也可一鍵切換雲端 LLM（DeepSeek / OpenAI / Google Gemini）。API Key 僅存於記憶體，關閉頁面即消失，不寫入任何檔案。適合金融機構、醫療機構、政府單位處理敏感數據。

### 信用風控建模全流程覆蓋
業界首個整合完整風控建模鏈的開源工具：
- **WOE/IV 分箱** → 變數篩選與預測力評估
- **12+ 種 ML 模型** → 包含 XGBoost、LightGBM、CatBoost、LDA
- **KS / Lift / Gain** → 模型區分力評估
- **SHAP 解釋** → 模型可解釋性（監管要求）
- **模型匯出** → .joblib 一鍵打包部署
- **PSI 監控** → 上線後資料漂移偵測

### Docker 一鍵部署，跨平台移植
`docker-compose up` 一行指令在任何 Windows / Mac / Linux 上啟動。無需手動安裝 Python、依賴、模型。團隊協作與機器遷移零障礙。

### 自然語言驅動分析
LangChain ReAct Agent 讓你用中文提問即可執行統計分析、假設檢定、機器學習。結合 RAG 知識庫，AI 助手能參考你上傳的公司文件回答問題。

### v4.0 新增功能
- **LLM 模型選擇器** — 側邊欄可即時切換地端/雲端模型
- **雲端 LLM 支援** — DeepSeek、OpenAI、Google Gemini 三大提供者
- **AI 變數描述** — 一鍵為所有欄位生成繁體中文 + 英文術語的業務描述
- **對話→資料字典** — 對話中的變數解釋可一鍵擷取並存入資料字典
- **對話匯出** — 匯出完整 AI 架構師對話紀錄 (Markdown 格式)
- **智慧分佈圖** — 根據資料特性自動推薦最適圖表類型，支援手動切換
- **CatBoost** — 原生支援類別特徵的梯度提升模型
- **LDA** — 線性判別分析（分類 + 降維）
- **Optuna 超參數調整** — 貝葉斯最佳化，比 GridSearchCV 更高效
- **演算法解說面板** — 每個演算法的原理、適用場景、不適用場景完整說明

### 多格式支援 + 中文友善
- 自動偵測 UTF-8 / Big5 編碼
- 支援 CSV、Excel、SAS、TXT 格式
- 匯出 CSV 使用 UTF-8 BOM，Windows Excel 直接開啟不亂碼
- 所有 UI 介面皆為繁體中文

### 模組化架構，易於擴展
8 個 Tab 各自獨立模組，新增功能只需加一個 `tabs/tab_xxx.py`，不影響現有程式碼。所有設定集中在 `config.py`，支援環境變數覆蓋。

### 3.1 下載模型

**方案 A — 32GB RAM / Mac M4 24GB（推薦）：**
```bash
ollama pull qwen2.5:14b
```

**方案 B — 16GB RAM：**
```bash
ollama pull qwen2.5:7b
```

**方案 C — 64GB RAM / Mac 32GB+：**
```bash
ollama pull qwen2.5:32b
```

> **Embedding 模型**: 本專案使用 `sentence-transformers` 自動下載 HuggingFace 模型（預設 `BAAI/bge-m3`），**不需要** 用 Ollama 下載 embedding 模型。首次啟動時會自動下載約 1.2GB。

### 3.2 模型比較

| LLM 模型 | 大小 | 中文能力 | 推理品質 | 適用場景 |
|----------|------|---------|---------|---------|
| qwen2.5:7b | 4.7 GB | 良好 | 一般 | 快速回應、輕量機器 |
| qwen2.5:14b | 9 GB | 優秀 | 良好 | **日常分析推薦** |
| qwen2.5:32b | 20 GB | 優秀 | 非常好 | 複雜推理、高階硬體 |

| Embedding 模型 | 大小 | 中文支援 | 備註 |
|----------------|------|---------|------|
| BAAI/bge-m3 | 1.2 GB | 優秀 | **預設推薦** (HuggingFace 自動下載) |
| all-MiniLM-L6-v2 | 80 MB | 較弱 | 英文為主，極輕量 |
| shibing624/text2vec-base-chinese | 400 MB | 優秀 | 中文專用替代 |

### 3.3 切換模型

**方式 A — 側邊欄即時切換（v4.0 推薦）：**

## 5. Docker 部署（推薦移植方式）

Docker 是**最簡單的移植方式**：只需安裝 Docker，一行指令即可在任何電腦上啟動。

### 5.1 安裝 Docker

前往 [Docker Desktop](https://www.docker.com/products/docker-desktop/) 下載安裝。

### 5.2 一鍵啟動

```bash
cd data_analysis_assistant
docker-compose up --build
```

首次執行會：
1. 建立應用程式容器（安裝所有 Python 依賴）
2. 啟動 Ollama 容器
3. 自動下載 LLM 模型（需等待）
4. 啟動 Streamlit 應用

完成後開啟 `http://localhost:8501`。

### 5.3 Docker 架構

```
┌─────────────────────────────────┐
│  docker-compose                 │
│  ┌────────────┐  ┌───────────┐ │
│  │ app        │  │ ollama    │ │
│  │ (Streamlit)│──│ (LLM)    │ │
│  │ :8501      │  │ :11434   │ │
│  └────────────┘  └───────────┘ │
│       │               │        │
│   hf_cache        ollama_data  │  ← Docker volumes (持久化)
│   ./storage                    │  ← RAG 知識庫 (本機目錄)
└─────────────────────────────────┘
```

### 5.4 常用 Docker 指令

| 指令 | 用途 |
|------|------|
| `docker-compose up -d` | 背景啟動 |
| `docker-compose down` | 停止服務 |
| `docker-compose logs -f app` | 查看應用程式日誌 |
| `docker-compose logs -f ollama` | 查看 Ollama 日誌 |
| `docker-compose up --build` | 重新建置並啟動 |

### 5.5 自訂環境變數

在 `docker-compose.yml` 的 `app.environment` 中加入：
```yaml
environment:
  - OLLAMA_BASE_URL=http://ollama:11434
  - LLM_MODEL=qwen2.5:7b          # 換模型
  - EMBED_MODEL=BAAI/bge-m3       # 換 embedding
  - OLLAMA_TIMEOUT=180             # 加長超時
```

### 5.6 移植到其他電腦

**步驟：**
1. 複製整個 `data_analysis_assistant/` 資料夾到目標電腦
2. 在目標電腦安裝 Docker Desktop
3. 開啟終端，`cd` 到資料夾
4. 執行 `docker-compose up --build`
5. 等待建置完成，開啟 `http://localhost:8501`

**需要的磁碟空間：** ~10-15 GB（含 Docker 映像 + LLM 模型）

## 8. Tab 2：數據預處理

### 缺失值處理策略

| 策略 | 說明 | 適用場景 |
|------|------|---------|
| 平均值 (mean) | 用該欄位平均值填充 | 數值型、近似常態分佈 |
| 中位數 (median) | 用中位數填充 | 數值型、有偏態或離群值 |
| 眾數 (mode) | 用出現最多的值填充 | 類別型變數 |
| 刪除 (drop) | 刪除含缺失值的列 | 缺失比例低 (<5%) |

### 資料型態轉換
- 數值轉換：文字 → 整數/浮點數
- 日期轉換：文字 → datetime
- 類別轉換：數值 → category

## 10. Tab 4：統計分析 + WOE/IV

### 假設檢定

| 方法 | 適用場景 |
|------|---------|
| 獨立樣本 t 檢定 | 比較兩組平均值 |
| 單樣本 t 檢定 | 對比理論均值 |
| ANOVA | 三組以上平均值比較 |
| 卡方檢定 | 類別變數獨立性 |
| 相關分析 | 變數關聯程度 (Pearson/Spearman/Kendall) |
| 線性迴歸 | 線性關係建模 |

### WOE/IV 分箱分析 (v3.0 新增)

**用途：** 信用風險建模中評估變數預測能力。

**操作步驟：**
1. 在分析類型選擇「WOE/IV 分析」
2. 選擇目標變數（需為 0/1 二元變數）
3. 選擇分箱方式：等頻 (quantile) / 決策樹 (tree) / 等寬 (equal_width)
4. 查看 IV 排名表 → 選擇單一變數查看 WOE 詳細

**IV 判讀標準：**

| IV 值 | 預測能力 |
|--------|---------|
| < 0.02 | 無預測力 |
| 0.02 ~ 0.1 | 弱 |
| 0.1 ~ 0.3 | 中等 |
| 0.3 ~ 0.5 | 強 |
| > 0.5 | 極強（可能過擬合） |

## 11. Tab 5：機器學習建模

### 任務類型

- **分類 (Classification)** — 預測離散類別（如：違約/未違約）
- **迴歸 (Regression)** — 預測連續數值（如：收入、分數）

### 模型清單

**分類：** Logistic Regression, Decision Tree, Random Forest, SVM, KNN, Naive Bayes, LDA, Gradient Boosting, XGBoost, LightGBM, CatBoost

**迴歸：** Linear Regression, Ridge, Lasso, Decision Tree, Random Forest, Gradient Boosting, XGBoost, LightGBM, CatBoost Regressor

### 運行模式

| 模式 | 說明 |
|------|------|
| 單模型訓練 | 選一個模型 → 訓練 → 查看結果 |
| 多模型比較 | 勾選多個模型 → 效能比較長條圖 |
| K-Fold 交叉驗證 | 設定折數 → 每折分數箱型圖 |
| 超參數調整 (GridSearchCV) | 網格搜索最佳參數 |
| 超參數調整 (Optuna) | **v4.0** 貝葉斯最佳化，更高效 |
| 📖 演算法解說 | **v4.0** 每個演算法的原理、適用/不適用場景 |

### 不平衡資料處理

- 無處理、SMOTE、ADASYN、Random Undersampling

### KS / Lift / Gain 風控指標 (v3.0 新增)

二分類模型訓練完成後，自動顯示：
- **KS 統計量**：衡量模型區分好壞客戶的能力（KS > 0.2 為可用）
- **Lift 圖**：前 N% 客戶中，模型捕捉的違約比例相對隨機的倍數
- **Gain 圖**：累積捕獲曲線

### SHAP 模型解釋 (v3.0 新增)

訓練完成後可展開「SHAP 模型解釋」區塊：
- **Summary Plot**：全局特徵重要性 + 方向
- 樹模型使用 `TreeExplainer`（速度快），其他模型用通用 `Explainer`
- 自動限制前 100 筆樣本以避免過慢

### 模型匯出 (v3.0 新增)

訓練完成後可下載 `.joblib` 檔案，包含：
- 訓練好的模型
- Scaler（如有）
- Label Encoder（如有）
- 特徵名稱列表
- 效能指標
- 模型元資料

載入方式：
```python
import joblib
bundle = joblib.load("model.joblib")

```

## 12. Tab 6：模型監控 (PSI) (v3.0 新增)

**PSI (Population Stability Index)** 用於監控模型上線後，輸入資料分佈是否漂移。

### 操作步驟

1. 先在 Tab 1 上傳**基準期資料**（如：訓練資料）
2. 切換到 Tab 6
3. 上傳**監控期資料**（如：最近一個月的新資料）
4. 系統自動計算共同數值欄位的 PSI

### PSI 判讀標準

| PSI 值 | 穩定度 | 建議 |
|--------|--------|------|
| < 0.1 | 穩定 | 無需動作 |
| 0.1 ~ 0.25 | 需關注 | 檢查變數分佈 |
| > 0.25 | 不穩定 | 需重新訓練模型 |

### 輸出內容

- PSI 報告表（各變數 PSI 值 + 穩定度標籤）
- PSI 長條圖（色彩標示穩定/不穩定）
- 分佈偏移圖（不穩定變數的基準 vs. 監控期直方圖）

## 18. FAQ 常見問答

### Q: 資料會上傳到雲端嗎？
**A**: 地端模式下不會，所有運算都在本機執行。雲端模式下，僅 AI 對話的文字提問會傳送至所選的雲端 LLM 提供者，原始資料不會上傳。

### Q: API Key 存在哪裡？複製資料夾會帶走嗎？
**A**: API Key **僅存於瀏覽器的 Session 記憶體**，關閉網頁即消失。不會寫入任何檔案（.env、config.py 等），複製資料夾到其他電腦**不會**帶走 API Key。每次開啟需重新輸入，或設定環境變數自動讀取。

### Q: 如何用環境變數設定 API Key（免每次輸入）？
**A**: 在系統環境變數中設定，例如：
- Windows: `setx DEEPSEEK_API_KEY "sk-xxxxx"`（需重啟終端）
- Mac/Linux: 在 `~/.bashrc` 加入 `export DEEPSEEK_API_KEY="sk-xxxxx"`
- 支援的環境變數：`DEEPSEEK_API_KEY`、`OPENAI_API_KEY`、`GEMINI_API_KEY`

### Q: Docker 和本機安裝的差別？
**A**: 功能完全一樣。Docker 的優勢是移植簡單（不需手動裝 Python/依賴），但需要額外的磁碟空間（Docker 映像約 3-5 GB）。

### Q: 如何把整個應用移到另一台電腦？
**A**:
- **Docker 方式**：複製資料夾 → 安裝 Docker → `docker-compose up --build`
- **本機方式**：複製資料夾 → 安裝 Python + Ollama → `pip install -r requirements.txt` → `streamlit run streamlit_app.py`

### Q: WOE/IV 是什麼？什麼時候該用？
**A**: WOE (Weight of Evidence) 和 IV (Information Value) 是信用風險建模的標準工具。用於評估各變數對預測目標（如違約）的區分能力。當你在做信用評分卡或風險模型時使用。

### Q: PSI 多久該監控一次？
**A**: 建議每月或每季監控一次。若 PSI > 0.25，表示資料分佈已顯著偏移，需重新訓練模型。

### Q: RAG 知識庫的資料儲存在哪？
**A**: 儲存在 `./storage/` 目錄中（ChromaDB 本機檔案）。移植時記得一起複製。

### Q: 為什麼推薦 qwen2.5？
**A**: Qwen2.5 是同級別中文能力最強的開源模型。也可使用 `llama3.1:8b`、`mistral:7b` 等替代。

### Q: SMOTE 是什麼？
**A**: 合成少數過採樣技術。當資料中某類別極少（如違約只有 5%），SMOTE 可改善模型對少數類的識別。

| 組件 | 技術 | 說明 |
